# 검증 메시지 UI 구현 가이드 — 분리선 행 · 볼드 체크

Flowit 채팅의 "워크플로우 생성/검증" AI 메시지를 **타임라인(분리선 행) + 볼드 체크** 형태로 구현하기 위한 명세입니다.
한 메시지가 두 상태를 가집니다.

1. **진행 중(스트리밍)** — 단계가 위에서 아래로 실시간 진행. 현재 단계만 문장이 흘러나옴(SSE).
2. **완료 + 검증 상세 보기 펼침** — 요약 한 줄 + `검증 상세 보기` 패널에 단계별 기록 보관.

## 2. 노드 마커 (볼드 체크) — 핵심

원(24×24) 안에 마커를 **flex 중앙 정렬**. 체크는 굵게(stroke 3.25), `display:block`으로 baseline 오프셋 제거.

```html
<!-- 완료: 갈색 원 + 볼드 흰 체크 -->
<span class="node node--done">
  <svg class="bold-check" viewBox="0 0 24 24"><polyline points="20 6 9 17 4 12"/></svg>
</span>

<!-- 진행 중: 코랄 보더 + 스피너 -->
<span class="node node--live"><span class="spinner"></span></span>

<!-- 대기: 빈 원 -->
<span class="node node--pending"></span>
```

```css
.node {
  width: 24px; height: 24px; border-radius: 9999px;
  display: flex; align-items: center; justify-content: center;  /* 중앙 정렬 */
  flex-shrink: 0; position: relative; z-index: 1; line-height: 0;
}
.node--done    { background: var(--color-accent); }
.node--live    { background: var(--color-coral-light); border: 2px solid var(--color-accent-coral); }
.node--pending { background: #fff; border: 2px solid var(--color-hl2); }

.bold-check {
  width: 15px; height: 15px; display: block;        /* baseline 오프셋 제거 */
  fill: none; stroke: #fff;
  stroke-width: 3.25;                                /* 볼드 */
  stroke-linecap: round; stroke-linejoin: round;
}

.spinner {
  width: 12px; height: 12px; border-radius: 50%;
  border: 2px solid var(--color-hl2);
  border-top-color: var(--color-accent-coral);
  animation: spin .7s linear infinite;
}
@keyframes spin { to { transform: rotate(360deg); } }

/* 진행 노드 펄스(선택) */
.node--live { animation: pulse 1.4s ease-out infinite; }
@keyframes pulse {
  0%,100% { box-shadow: 0 0 0 0 rgba(232,148,92,.45); }
  50%     { box-shadow: 0 0 0 5px rgba(232,148,92,0); }
}
```

> **중앙 정렬 포인트:** 원에 `display:flex; align-items:center; justify-content:center`, 체크 `<svg>`에 `display:block`. lucide 아이콘을 쓰면 인라인 baseline 때문에 살짝 내려가므로 `display:block` 필수.

## 3. 타임라인 레일

각 행은 `[노드][내용]` 가로 배치, 노드 중심을 지나는 세로선이 레일.

```css
.timeline { position: relative; }
.timeline::before {            /* 세로 레일 */
  content: ""; position: absolute;
  left: 11px;                  /* 노드(24px) 중심 = 12px, 선두께 2px → 11px */
  top: 10px; bottom: 10px;
  width: 2px; background: var(--color-hl2);
}
.t-row { display: flex; align-items: flex-start; gap: 14px; position: relative; }
.t-row + .t-row { margin-top: 18px; }   /* 진행 중 */
/* 상세에서는 행 간격을 24px로 넓혀 가독성↑ */
```

## 4. 진행 중(스트리밍) 영역

```
[● 진행 중 · 스트리밍]            ← 캡션(코랄 점 + 대문자)
🦦 워크플로우를 만들고 있어요       ← 굵은 제목

✔ 의도 분석
   주간 리포트 자동화로 파악했어요.
✔ 노드 선출
   후보 11개 중 3개를 선정했어요.
◐ 워크플로우 작성                 ← 진행 노드(스피너)
   노드 3개를 2개 연결로 잇는 중▍   ← 커서 깜빡임
○ 품질 평가
   대기 중                        ← ink4
```

- 완료 단계: 제목 `ink`(700) + 설명 `ink3`(13px).
- 진행 단계: 설명 끝에 깜빡이는 커서 `▍`(코랄). SSE 토큰이 이 텍스트에 append.
- 대기 단계: 제목·설명 모두 `ink4`.

```css
.cursor::after { content:"▍"; color: var(--color-accent-coral);
  margin-left:1px; animation: blink .9s step-start infinite; }
@keyframes blink { 50% { opacity: 0; } }
```

**SSE 매핑:** 서버 이벤트 → 해당 단계 노드 상태 전이.
`pending → live`(스피너) → 텍스트 토큰 스트리밍 → `live → done`(볼드 체크 + 요약 한 줄 확정) → 다음 단계 `live`.

## 5. 완료 + 검증 상세 보기

```
[✔ 완료 · 검증 상세 보기 펼침]
🦦 검증을 모두 통과했어요. 4단계를 차례로 점검했고 품질 점수는 10.0 / 10입니다.

┌ 🗂 검증 상세 보기                          ▲ ┐   ← 토글 헤더
│  ✔ 의도 분석                                  │
│      요청 유형   새 워크플로우 생성             │
│      원문 요청   매주 월요일 9시에 …           │
│      추출 정보   트리거 매주 월 09:00 · …      │
│  ✔ 노드 선출                                  │
│      후보       11개 검색                      │
│      선정       메일 수신 · 내용 요약 · …      │
│  ✔ 워크플로우 작성  …                          │
│  ✔ 품질 평가     …                            │
└──────────────────────────────────────────────┘

이 워크플로우는 메일 읽기와 슬랙 쓰기 권한이 필요하며, 위험도는 보통입니다.
[ 저장하고 활성화 ]  [ 편집 ]
```

### 5-1. 상세 패널(disclosure)

```css
.detail { background:#fff; border:1px solid var(--color-line-soft); border-radius:16px; overflow:hidden; }
.detail__head { display:flex; align-items:center; gap:8px;
  padding:12px 16px; border-bottom:1px solid var(--color-line-soft); background: rgba(251,248,242,.6);
  font-size:12.5px; font-weight:700; color: var(--color-ink2); }
.detail__body { padding:18px 16px; }
```

### 5-2. 단계별 "분리선 행" (라벨 좌 / 값 우)

각 단계 노드 아래에 key-value 행. 행마다 얇은 분리선.

```html
<div class="t-row">
  <span class="node node--done"><svg class="bold-check">…</svg></span>
  <div class="t-body">
    <div class="t-title">의도 분석</div>
    <div class="kv">
      <div class="kv__row"><div class="kv__k">요청 유형</div><div class="kv__v">새 워크플로우 생성</div></div>
      <div class="kv__row"><div class="kv__k">원문 요청</div><div class="kv__v">매주 월요일 9시에 광고 시트를 읽어 요약하고 슬랙으로 보내줘</div></div>
      <div class="kv__row"><div class="kv__k">추출 정보</div><div class="kv__v">트리거 매주 월 09:00 · 대상 #ad-report</div></div>
    </div>
  </div>
</div>
```

```css
.t-title { font-size:14px; font-weight:700; color: var(--color-ink);
  white-space:nowrap; margin-bottom:8px; }
.kv__row { display:flex; gap:12px; padding:8px 0;
  border-top:1px solid rgba(236,227,214,.7); }
.kv__row:first-child { border-top:0; }
.kv__k { width:84px; flex-shrink:0; font-size:12px; font-weight:700;
  color: var(--color-ink3); white-space:nowrap; }
.kv__v { flex:1; font-size:13px; font-weight:500; color: var(--color-ink);

```

## 7. 데이터 모델 (구현 참고)

```ts
type StepState = "pending" | "live" | "done";

interface Step {
  id: "intent" | "select" | "build" | "qa";
  title: string;          // "의도 분석"
  state: StepState;
  summary?: string;       // 완료 후 한 줄 (진행 중 스트리밍 → 확정)
  detail: { label: string; value: string }[];  // 검증 상세 보기 내용
}
```

- **진행 중** 렌더: `step.state`로 노드 모양 결정, `live` 단계만 `summary`에 SSE 토큰 append + 커서.
- **완료** 렌더: 모든 노드 `done`(볼드 체크), `detail` 배열을 분리선 행으로 출력.
- 즉 **검증 상세 보기 = 스트리밍 때 흘러간 기록을 `detail`로 접어둔 것**.
